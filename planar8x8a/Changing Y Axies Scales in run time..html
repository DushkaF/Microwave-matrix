The upper Range property is a subproperty of Marks, i.e. it determines the range which is labelled by ticks and marks - you see that when you scroll up a bit. This feature is useful when you overlay several axes on the same line and offset them by means of AutoscaleTransformations (like in the tutorial http://wiki.lazarus.freepascal.org/TAChart_Tutorial:_Multiple_Panes_in_one_Chart).

The other Range property is a direct property of the axis and determines axis scaling, i.e. the value corresponding to the upper and lower end of the axis.

In order to change the color of an axis you have to modify several properties:
  • AxisPen.Color AND AxisPen.Visible for the axis line (yes - the axis line is hidden by default. The box that you see around the chart area is the "Frame").
  • Marks.LabelFont.Color for the axis labels
  • TickColor for the axis ticks. If you have a minor axis, don't forget the Minors[].TickColor
  • Title.LabelFont.Color for the axis caption. Again you have to turn on Title.Visible to show the caption.

Sorry I did not see this one, but I think it is not good to leave it without an answer...

Having such irregular labels at the right axis is not normal. You probably fiddled around with the Intervals property and forgot something to set back to its original value. The default settings of Intervals are:
  • Count = 5
  • MaxLength = 50
  • MinLength = 10
  • NiceSteps = 0.2|0.5|1.0
  • Options = [aipUseMaxLength,aipUseMinLength,aipUseNiceSteps]
  • Tolerance = 0
NiceSteps means that the label-finding algorithms tries to set the label marks at the values shown, i.e., 0.01, 0.02, 0.05, 0.1, 0.2, 0.5, 1, 2, 5, 10 etc. For this to work the corresponding option must be set.
If you have aipUseCount in the Options the labeller will put priority on the label count which normally produces weird label values, like in your case.
If labels overlap you may increase the MinLength and MaxLength values - these parameters (along with their corresponding options) control the density of labels along the axis.
Tolerance helps in connection with rounding errors, I often get "nicer" labels when this property is increase moderately (such as 3 or 5).

Quote
Is there somewhere that lists all the properties of the Chart component and their uses?
In addition to the tutorials, there is the TAChart documentation and the TAChart User Guide. Moreover, don't forget the bunch of demo projects in the folder components/tachart/demo of your Lazarus installation (they are described in http://wiki.lazarus.freepascal.org/TAChart_Demos).
